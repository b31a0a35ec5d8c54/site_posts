Этот проект был создан в рамках тестового задания.
Ниже приведено само задание в исходном виде

```
Тестовое задание для разработчика Python
● Нужно написать JSON API на основе Django REST framework.
● Авторизация для доступа ко всем API Endpoint – по логин/паролю,
● новый пользователь API может быть добавлен/удален только через
django admin.
● Каждый пользователь может делать CRUD операции по своим постам,
● смотреть список пользователей в системе и их постов, а также
Версия Django >= 1.11, Python3.4
Необходима минимальная обработка ошибок API – валидация наличия
обязательных полей в POST запросах, проверка существования post_id
при редактировании/удалении.
После выполнения нужно указать, сколько времени заняло выполнение.
Код по тестовому заданию можно опубликовать в своем портфолио
(github, bitbucket).
Список endpoint API
1) GET /users
возвращает JSON список пользователей в системе, поля id, username,
registered_at, posts_number (общее число постов), posts_per_day (среднее
        число постов в день)
GET-параметры для фильтрации /users : username (точное совпадение),
posts_number (общее число постов >= N)
2) GET /posts
возвращает JSON список всех постов в системе, поля id, user_id,
title, body.
GET-параметры для фильтрации /posts : username (точное совпадение,
        вернуть только посты пользователя username)
3) CRUD для постов пользователя:
GET /my_posts - вернуть свои постыPOST /my_posts - создать новый пост с обязательными полями title,
body.
PUT /my_posts/[id] - изменить существующий пост
DELETE /my_posts/[id] - удалить пост
4) Желательно написать необходимые тесты для полученного API.

```
